## Windows SmartScreenが警告を出す

**症状**: インストーラー起動時に「Windows によって PC が保護されました」ダイアログが出て青いブロック画面になる。

**原因**: 現状インストーラーは Authenticode 署名を持たないため、Microsoft の
評価データベースに未登録扱いになる。悪意のあるファイルというわけではありません。

**対処**:
1. ダイアログの **「詳細情報」** をクリック
2. 表示された「発行元不明」の下に **「実行」** ボタンが出るので押す

同じインストーラーを使い続けていれば、SmartScreen が「知られたファイル」と
学習するので警告は自然と減ります(数十〜数百ダウンロードで消える)。

将来的には Authenticode 署名を導入予定。

## `0416:5408` が認識されない / フレームが送れない

**症状**: ヘッダの `LCD` ピルが `disconnected` のまま。`backend.log` に
`Failed to set USB configuration` や `Device 0416:5408 not found` が出る。

**原因**: `0416:5408` は Windows 標準では純正 TRCC ドライバにバインドされていて、
pyusb が触れない状態です。**Zadig で WinUSB へ差し替える**必要があります。

**対処**:
1. [Zadig](https://zadig.akeo.ie/) を起動
2. メニュー **Options → List All Devices**
3. リストから `USBDISPLAY` (VID `0416` / PID `5408`) を選択
4. Driver に **WinUSB** を選択して **Replace Driver**
5. Trofeo Vision Studio を再起動 (トレイから Quit → 再ログオン、もしくは
   タスクマネージャーから終了)

> **注意**: この操作で **純正 TRCC は使えなくなります**。純正に戻すには
> デバイスマネージャー → 該当 `USBDISPLAY` を右クリック → **デバイスの
> アンインストール** → 「ドライバーソフトウェアを削除する」にチェック → 削除。
> USBを抜き差しすると純正ドライバが再自動導入されます。

## CPU温度が `-- °C` のまま

**症状**: 他のセンサー(GPU温度 / 各種負荷 / RAM)は出るのに CPU 温度だけ
表示されない。

**原因**: LibreHardwareMonitor が CPU 温度を読むには **管理者権限で動作
する ring0 ドライバ** (WinRing0 相当) が必要です。管理者昇格していない
バックエンドは他のセンサーは読めても CPU 温度は読めません。

**対処**:
- **常駐運用**(推奨): インストーラーが登録する管理者スケジュールタスク
  `TrofeoVisionStudio` がログオン時に昇格プロセスとして起動するので、
  再起動後は自動で読めるようになります。**Trofeo Vision Studio が最初から
  管理者で走ってなかった** ケースは、一度 UAC 付きで起動し直せば
  `ensureAutostartTask()` がタスクを登録し直します。
- **開発モード**: `python server.py` を **管理者 PowerShell** で起動。

タスクマネージャー → 詳細タブ → `server.exe` を確認。**「昇格」列が「はい」**
になっているのが正常。

## メモリ整合性 (HVCI) 有効環境でCPU温度が読めない

**症状**: 管理者で起動しているのに CPU 温度が `-- °C`。`backend.log` に
`WinRing0` 系のエラーや `driver signing` に関するエラー。

**原因**: Windows 11 の **メモリ整合性 (Memory Integrity / HVCI)** を有効に
していると、LHM 内蔵の古い署名の WinRing0 ドライバが読み込めません。

**対処**: **HVCI を無効化する必要はありません**。代わりに **PawnIO** を
利用します。PawnIO は Microsoft 認証済みの汎用ドライバで、HVCI 有効環境でも
ring0 レジスタアクセスができます。

- インストーラーは **PawnIO_setup.exe を silent install** します(通常は自動)
- 手動確認: `C:\Program Files\PawnIO\` が存在すればOK
- 手動インストール: [PawnIO 公式](https://pawnio.eu/) から `PawnIO_setup.exe`
  を入手して実行

PawnIO がインストール済みの環境では、LHM は自動でそちらを優先します。

## ログオン時に自動起動しない

**症状**: 再起動しても常駐が立ち上がらない。

**原因**: スケジュールタスク `TrofeoVisionStudio` の登録が壊れている
可能性があります。

**対処**:
1. スタートメニューから `taskschd.msc` で **タスク スケジューラ** を起動
2. **タスク スケジューラ ライブラリ** の直下に `TrofeoVisionStudio` が
   あることを確認 (無い場合は 4 へ)
3. **状態** が「準備完了」で **トリガー** が「ログオン時」になっていることを確認
4. Trofeo Vision Studio を **UAC付きで起動**(スタートメニューから
   右クリック → 管理者として実行)。`ensureAutostartTask()` がタスクを
   登録し直します。 `backend.log` の隣にある `main` プロセスのログで
   `autostart task registered/repaired` が出れば成功

それでも直らない場合はインストーラーを **上書き実行** してください
(1.8.0以降は自動アップデートで済むが、初回は手動)。

## LCD が固まった / フレームが更新されない

**症状**: LCD が古い画像のまま更新されない。ヘッダの `out X fps` が 0 か
異常に低い。

**確認手順**:
1. トレイの Trofeo Vision Studio を右クリック → **Show editor**
2. ヘッダの `Backend` ピル、`LCD` ピルが両方 **緑 / online** か
3. 緑でない場合: `Backend offline` なら server.exe が死んでる。
   1.8.0+ なら自動で 1-30秒 後に自動リスタート
4. `LCD disconnected` なら USB 側で問題。Zadigの WinUSB バインドが
   外れていないか確認。USBケーブル抜き差しでも直ることが多い

**バックエンドが暴走している場合の緊急対処**:
- タスクマネージャー → `server.exe` を強制終了
- 1.8.0+ は Electron 側から自動で新しいのを起動します
- タスクマネージャー → `Trofeo Vision Studio.exe` も終了する場合はトレイの
  Quit から抜けるのが安全(LCD blanking → 綺麗に終了)

## オーディオビジュアライザーが反応しない

**症状**: ビジュアライザー ウィジェットが無反応 / バーが動かない。

**原因**: WASAPI ループバックキャプチャが起動できない状況。多くは、
「デフォルトオーディオ出力デバイスがサイレント / 存在しない」ケース。

**対処**:
1. Windows の **サウンド設定** で **出力** に有効なデバイス(ヘッドフォン /
   スピーカー / HDMI モニタ音声など)が選ばれているか確認
2. 音声を再生する — 何か音が鳴っていないと当然反応しません
3. `backend.log` の `[audio]` 行を確認:
   - `capture opened: <デバイス名>` が出れば正常
   - `capture failed:` があるとキャプチャエラー(v1.6.2+ は 30秒後に自動再試行)

**デバイスを切り替えたら反応が止まる** 場合、30秒後に自動でリオープンするか
待ってください(PortAudio のデバイス一覧を再取得する仕様)。

## 自動アップデートが動かない

**症状**: 1.8.0+ をインストールしたのに、新しいバージョンが出てもトレイに
「Install update X.Y.Z」が出ない。

**確認**:
1. トレイ → **Check for updates** を手動で押してみる
2. `%APPDATA%\Trofeo Vision Studio\` 配下の `main` プロセスログに
   `[updater]` エントリがあるか

**原因**:
- **リポジトリが private の場合**、アップデーターは GH トークンがないと
  アクセス失敗します(このリポジトリは 2026-07-05 に public 化済みなので
  1.8.0以降は問題ないはず)
- **バージョンがすでに最新の場合** は当然何も起こりません
- **ネットワークプロキシ経由** の場合、`api.github.com` に到達できない可能性

**手動アップデート**: [Releases](https://github.com/Nekunegi/TrofeoVisionStudio/releases)
から新しいインストーラーをダウンロードして実行すれば強制的に最新化されます。

## ログの場所

| ファイル | 場所 | 内容 |
|---------|------|-----|
| `backend.log` | `%APPDATA%\Trofeo Vision Studio\backend.log` | Python バックエンドの stdout / stderr。USB / センサー / 音声 / 通知の状況 |
| `renderer.log` | `%APPDATA%\Trofeo Vision Studio\renderer.log` | Electron レンダラの console.log。ダッシュボード側のエラー |

いずれも **5MB を超えると自動で切り詰め**られます。

問題を報告する際はこの 2 つを添付してもらえるとありがたいです。
